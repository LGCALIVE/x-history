# X History

[English](README.md) | **中文**

一个轻量级 Chrome 浏览器插件，自动记录你在 X（Twitter）上阅读过的推文。再也不会丢失看过的内容。

## 功能特点

- **自动记录** — 点进任何推文即自动保存，无需手动操作
- **搜索** — 按作者名、@ID 或推文内容搜索
- **紧凑/全文模式** — 一键切换列表浏览和全文预览
- **日期分组** — 按天分组（今天、昨天、具体日期），快速定位
- **头像显示** — 显示博主头像，方便识别
- **清理缓存** — 一键清除所有数据，释放存储空间
- **隐私优先** — 所有数据仅存本地，不发送任何网络请求，不追踪用户

## 安装教程

### 方式一：下载 ZIP（最简单）

1. 点击本页面顶部绿色的 **Code** 按钮
2. 选择 **Download ZIP**
3. 解压下载的文件到任意位置
4. 打开 Chrome 浏览器，地址栏输入 `chrome://extensions` 回车
5. 打开右上角的 **开发者模式** 开关
6. 点击左上角 **加载已解压的扩展程序**
7. 选择刚才解压出来的 `x-timeline-history-main` 文件夹
8. 完成！工具栏会出现 X History 图标

### 方式二：Git 克隆

```bash
git clone https://github.com/LGCALIVE/x-timeline-history.git
```

然后按照上面第 4-8 步操作。

## 使用教程

### 第一步：记录推文

安装完成后，**什么都不用设置**。

1. 正常打开 X（x.com 或 twitter.com）
2. 正常浏览你的时间线
3. **点进任何一条推文**查看详情
4. 这条推文就自动被记录了

> 注意：只有你**点进帖子详情页**才会记录，仅仅在时间线上滑过的内容不会被记录。

### 第二步：查看浏览记录

1. 点击 Chrome 工具栏右上角的 **X History 图标**
2. 弹出窗口中显示你的阅读记录，最近看的排在最前面
3. 记录按日期分组：今天、昨天、更早的日期

### 第三步：搜索

在搜索框中输入关键词，可以搜索：
- 推文内容中的文字
- 作者的显示名称
- 作者的 @ID

### 第四步：切换视图

点击状态栏右边的切换按钮：
- **紧凑模式** — 每条推文只显示一行摘要，方便快速浏览大量记录
- **全文模式** — 显示完整推文内容和图片

### 第五步：打开原帖

在列表中点击任何一条推文，会在新标签页中打开 X 上的原始帖子。

### 第六步：清理数据

点击右上角的 **Clear** 按钮，确认后清除所有保存的推文，释放本地存储空间。

## 工作原理

插件在 X 的网页中运行一个内容脚本。当你浏览推文详情页（URL 包含 `/status/`）时，脚本从页面 DOM 中读取推文内容（作者、正文、图片、时间），保存到 Chrome 本地存储中。

**不调用任何 API，不连接外部服务器，数据不离开你的浏览器。**

## 权限说明

| 权限 | 用途 |
|---|---|
| `storage` | 本地保存推文数据 |
| `unlimitedStorage` | 支持最多保存 10,000 条推文 |
| 内容脚本（仅 x.com / twitter.com） | 从页面读取推文内容 |

不需要 `tabs`、`webRequest` 等敏感权限。

## 存储占用

- 每条推文约 1 KB（图片仅保存 URL，不缓存图片本身）
- 10,000 条推文约 10 MB
- 数据完全存储在本地

## 技术栈

- 原生 JavaScript + CSS，零外部依赖
- Chrome Extension Manifest V3
- 插件总体积约 50 KB
